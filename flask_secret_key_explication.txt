===============================================================================
                        FLASK SECRET KEY : RÔLE ET FONCTIONNEMENT
===============================================================================

QU'EST-CE QUE LA FLASK SECRET KEY ?
===============================================================================

La FLASK_SECRET_KEY est une chaîne de caractères secrète utilisée par Flask pour 
sécuriser les données côté client. Elle sert de "clé de chiffrement" pour 
protéger les informations sensibles échangées entre le serveur et le navigateur.

Dans votre .env :
FLASK_SECRET_KEY=your-secret-key-here

===============================================================================
À QUOI SERT LA SECRET KEY ?
===============================================================================

1. SÉCURISATION DES SESSIONS
----------------------------
Flask utilise des "sessions" pour stocker des données temporaires pour chaque 
utilisateur (comme "utilisateur connecté", panier d'achat, préférences, etc.).

Sans secret key : Les sessions sont stockées en clair → DANGER
Avec secret key : Les sessions sont chiffrées → SÉCURISÉ

Exemple :
- Utilisateur se connecte → Flask stocke "user_id=123" dans la session
- Sans clé : Cookie = "user_id=123" (visible et modifiable)
- Avec clé : Cookie = "eyJ1c2VyX2lkIjoxMjN9.XYZ..." (chiffré et protégé)

2. PROTECTION CSRF (Cross-Site Request Forgery)
-----------------------------------------------
La secret key sert à générer des tokens CSRF qui protègent contre les attaques 
où un site malveillant tente d'exécuter des actions sur votre application.

Exemple d'attaque sans protection :
- Site malveillant : <form action="http://localhost:5000/api/documents" method="POST">
- L'utilisateur clique → Document créé à son insu

Avec secret key :
- Flask génère un token unique : csrf_token="abc123xyz"
- Seules les requêtes avec le bon token sont acceptées
- Site malveillant ne peut pas deviner le token → Attaque bloquée

3. SIGNATURE DES COOKIES
------------------------
Flask signe tous les cookies avec la secret key pour garantir leur intégrité.

Sans signature : Cookie peut être modifié par l'utilisateur
Avec signature : Toute modification invalide le cookie

4. CHIFFREMENT DES DONNÉES SENSIBLES
------------------------------------
Si votre application stocke des données sensibles temporaires, Flask peut les 
chiffrer avec la secret key.

===============================================================================
COMMENT ÇA FONCTIONNE TECHNIQUEMENT ?
===============================================================================

1. ALGORITHME DE CHIFFREMENT
----------------------------
Flask utilise l'algorithme HMAC-SHA1 ou HMAC-SHA256 avec votre secret key.

Processus :
Données originales + Secret Key → Algorithme HMAC → Données chiffrées + Signature

Exemple simplifié :
- Données : {"user_id": 123}
- Secret Key : "your-secret-key-here"
- Résultat : "eyJ1c2VyX2lkIjoxMjN9.YbCdEf.GhIjKlMnOpQrStUvWxYz"
  ↑ Données encodées  ↑ Timestamp  ↑ Signature HMAC

2. VÉRIFICATION D'INTÉGRITÉ
---------------------------
Quand Flask reçoit un cookie :
1. Sépare les données de la signature
2. Recalcule la signature avec la secret key
3. Compare avec la signature reçue
4. Si identique → Cookie valide
5. Si différent → Cookie rejeté (potentiellement modifié)

3. EXPIRATION AUTOMATIQUE
-------------------------
Flask peut inclure un timestamp dans les cookies signés pour les faire expirer 
automatiquement après un certain temps.

===============================================================================
DANS VOTRE APPLICATION QR ARCHIVES
===============================================================================

UTILISATION ACTUELLE :
----------------------
Votre application utilise la secret key pour :
- Sécuriser les sessions Flask (même si pas de login pour l'instant)
- Protéger les formulaires web contre CSRF
- Signer les cookies de session

EXEMPLE CONCRET :
-----------------
Quand un utilisateur visite http://localhost:5000 :
1. Flask crée une session vide
2. Génère un cookie de session signé avec votre secret key
3. Envoie le cookie au navigateur
4. Le navigateur renvoie le cookie à chaque requête
5. Flask vérifie la signature pour s'assurer qu'il n'a pas été modifié

CODE DANS app.py :
------------------
app.secret_key = os.environ.get('FLASK_SECRET_KEY', 'dev-key-not-secure')

Cette ligne dit à Flask d'utiliser la clé depuis .env, ou 'dev-key-not-secure' 
par défaut (uniquement pour développement).

===============================================================================
SÉCURITÉ ET BONNES PRATIQUES
===============================================================================

1. GÉNÉRATION D'UNE CLÉ SÉCURISÉE
---------------------------------
Votre clé actuelle : "your-secret-key-here" → PAS SÉCURISÉE

Bonne clé (exemple) :
FLASK_SECRET_KEY=8f42a73054b1749e7e8b8a9d4c6e2f1a3b5d7c9e0f2a4b6c8d0e1f3a5b7c9d1e

Génération recommandée :
- Python : import secrets; secrets.token_hex(32)
- Ligne de commande : openssl rand -hex 32
- Minimum 32 caractères, mélange lettres/chiffres

2. STOCKAGE SÉCURISÉ
--------------------
✅ Dans .env (comme vous faites)
✅ Variables d'environnement système
❌ Jamais dans le code source
❌ Jamais dans un fichier versionné (git)

3. ROTATION DE LA CLÉ
---------------------
En production, changez la secret key régulièrement :
- Tous les 6-12 mois minimum
- Immédiatement si suspicion de compromission
- Attention : changer la clé invalide toutes les sessions existantes

===============================================================================
QUE SE PASSE-T-IL SANS SECRET KEY ?
===============================================================================

ERREUR FLASK :
--------------
RuntimeError: The session is unavailable because no secret key was set.

CONSÉQUENCES :
--------------
- Impossible d'utiliser les sessions Flask
- Pas de protection CSRF
- Cookies non signés → facilement modifiables
- Application vulnérable aux attaques

===============================================================================
EXEMPLE PRATIQUE DE GÉNÉRATION
===============================================================================

Pour générer une nouvelle clé sécurisée :

1. PYTHON :
-----------
import secrets
print(secrets.token_hex(32))
# Résultat : 8f42a73054b1749e7e8b8a9d4c6e2f1a3b5d7c9e0f2a4b6c8d0e1f3a5b7c9d1e

2. POWERSHELL :
---------------
[System.Web.Security.Membership]::GeneratePassword(64, 0)
# Résultat : aB3dE6fG9hI2jK5lM8nO1pQ4rS7tU0vW3xY6zA9bC2dE5fG8hI1jK4lM7nO0pQ3r

3. LIGNE DE COMMANDE (si OpenSSL installé) :
---------------------------------------------
openssl rand -hex 32
# Résultat : 7c8e9f1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e
